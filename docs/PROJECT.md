# ArchLens 项目说明

## 我们是谁

ArchLens / 建筑透镜是一个开源的建筑、城市与规划案例知识入口。它参考 Mobbin 的浏览效率，但不复制 Mobbin 的内容或品牌：我们把“界面案例库”替换成“可追溯的建筑研究资料库”。

## 我们的理念

### 先理解，再生成

案例不是可以随意消费的图片。每个判断都应该有原始资料、上下文或明确的编辑说明作为依据。

### 把案例当作方法

我们不只收集“长什么样”，还记录项目如何回应场地、功能、公共性、材料、气候和建造风险。

### AI 是入口，不是替代品

ArchLens 不绑定模型，也不提供官方实时 AI 助手。我们通过结构化资料和 MCP，把可靠的上下文交给用户自己的 Agent。

### 内容生产应该可复用

案例生产不是一次性编辑工作，而是一套可以由贡献者重复执行、检查和改进的 Skill。

## 我们的任务

1. 建立有来源、有结构、有局限说明的案例库。
2. 让用户能在三分钟内从一个短 Prompt 开始研究。
3. 提供可下载、可引用、可继续处理的研究资料包。
4. 让外部 Agent 通过 MCP 读取同一份结构化案例知识。
5. 用 GitHub、Skill 和 CI 建立开放协作的内容生产机制。

## 我们的预期

- 对设计师：缩短找案例和建立论据的时间。
- 对研究者：让跨项目比较和资料引用更可控。
- 对 Agent 开发者：提供稳定、开放、可组合的建筑知识上下文。
- 对社区：让案例的选择、解释和修订过程可以被看见。

## 非目标

- 不在首版生成完整建筑方案。
- 不把 AI 生成图当作真实项目资料。
- 不在没有来源和许可说明的情况下批量抓取或再分发内容。
- 不用登录、社交功能和复杂后台掩盖内容质量问题。
