# 系列结构库（5 种主结构 + 选型指南）

## 1. 渐进深入型（最常用 / 最稳）

**结构**：从概念 → 拆解 → 应用 → 案例 → 误区 → 工具 → 自测 → 总结，层层递进。

**适用**：
- 拆书系列（深度工作、原则、非暴力沟通）
- 知识科普（心理学、营销学、宏观经济）
- 方法论连载（GTD、番茄工作法、第二大脑）

**骨架模板（8 期版）**：

| Vol | 副标题模板 | 核心动作 |
|---|---|---|
| 1 | 一个被忽视的 X | 概念破冰，给"为什么这事关你" |
| 2 | 拆解 X 的 3 层结构 | 拆骨，把复杂事物讲明白 |
| 3 | 90% 的人都做错的 X 误区 | 反常识打脸，建立信任 |
| 4 | 顶尖高手的 X 习惯 | 案例 / 故事，建立向往 |
| 5 | X 的公式 | 给可记忆的公式/口诀 |
| 6 | 30 分钟上手的 X 工具 | 给可执行方法 |
| 7 | 自测：你处在 X 的哪一级 | 互动钩（评论刷起来） |
| 8 | 一份完整的 X 行动清单 | 总结 + 系列导览 |

**节奏建议**：周 1 期，总周期 8 周。

**避坑**：
- Vol.3 不要和 Vol.1 撞（都讲"误区"会重复）
- Vol.7 是互动高点，必须预留评论钩

---

## 2. 并列主题型（灵活 / 易扩展）

**结构**：每期一个独立切面，互不依赖，可任意顺序读。系列感靠"统一母题 + 统一格式"。

**适用**：
- 书单系列（10 本不同的书，每本 1 期）
- 工具合集（10 个 App，每期 1 个）
- 案例集（N 个真实故事，每期 1 个）

**骨架模板（N 期版）**：

```markdown
Vol.{N} · {切面名}
- 本期主角：{书/工具/人}
- 一个关键数据/事实
- 三个可用要点
- 一句可引用的金句
- 与系列的连接：{这个切面在系列里的位置}
```

**节奏建议**：周 2 期或隔日更，适合快速涨粉期。

**避坑**：
- 必须有"统一格式"（如"3 个要点 + 1 个金句"），避免风格漂移
- 母题必须强（不然用户分不清是不是同系列）

---

## 3. 故事线型（涨粉最快 / 难持续）

**结构**：作者本人做一件事的连载，过程即内容（30 天挑战、90 天转型、一年读 100 本书）。

**适用**：
- 转型/创业 vlog
- 学习挑战
- 健康/习惯改造

**骨架模板（12–30 期）**：

```markdown
Vol.{N} · Day {D}
- 今天做了什么
- 遇到了什么困难
- 一个小顿悟
- 明日预告
- 数据变化（粉丝、体重、收入等）
```

**节奏建议**：日更或隔日更。

**避坑**：
- 30 天后必须设"中点转折"，不然易疲
- 真实 > 完美，掉量要承认
- 不要超过 60 期，人会崩

---

## 4. 挑战体型（高互动 / 强 IP）

**结构**：作者邀请读者一起做挑战，每期 1 个 action，下期公布结果 + 新 action。

**适用**：
- 21 天早起
- 7 天极简
- 30 天读书

**骨架模板（21–30 期）**：

```markdown
Vol.{N} · 第 {D} 天挑战
- 本期 action：{具体到分钟}
- 上期战报：{多少人完成 / 我自己完成度}
- 本期彩蛋：{1 个小奖励/惊喜}
- 评论区钩：{让大家报进度}
```

**节奏建议**：日更。

**避坑**：
- Action 必须 ≤ 10 分钟（用户嫌麻烦就放弃）
- 战报必须真实（哪怕完成度低）

---

## 5. 问答体（最适合冷启动后 / 中腰部）

**结构**：Vol.N = 解答读者提的 N 个问题，问题来源 = 评论区 + 私信 + 选题策划历史。

**适用**：
- 中腰部账号，已有固定读者
- 想强化"互动感"
- 想清理历史遗留 FAQ

**骨架模板（N 期）**：

```markdown
Vol.{N} · 你问我答
- 问题 1：{来自 XX 粉丝}
- 问题 2：{来自评论区}
- 问题 3：{来自私信}
- 我的回答
- 加餐 1 句话
```

**节奏建议**：周 1–2 期，夹在主系列中间做"互动润滑剂"。

**避坑**：
- 问题必须真来自读者，不允许 AI 模拟
- 至少留 1 个"自问自答"型钩（拉动新读者）

---

## 选型决策树

```
用户需求
  ├── 拆书 / 科普 / 方法论 → 渐进深入
  ├── 工具 / 书单 / 案例合集 → 并列主题
  ├── 转型 / 创业 / 改造 → 故事线
  ├── 想做挑战拉互动 → 挑战体
  ├── 已有人设想互动 → 问答体
  └── 不确定 → 渐进深入（最稳，事后可调）
```

## 组合用法

真实账号常混搭：

- 主结构 = 渐进深入（8 期大骨架）
- Vol.4 后插 1 期问答体（润滑）
- Vol.6 后插 1 期挑战体（拉互动）
- Vol.8 收尾 → 下一个并列主题系列

### 复用指南

LLM 在 `/系列化规划` 排期时，模仿上述 5 个 Worked 表的格式生成新主题的 Vol.1-N 表。结构、字段、调色板继承模式都从 Worked 例子直接读取，**不**重新发明。

> **L32 区分说明**：
> - **`series-structure` 5 类**（渐进深入 / 并列主题 / 故事线 / 挑战体 / 问答体）：是系列的**整体骨架**（决定 Vol.1-N 的宏观类型）
> - **`relation_to_previous` 4 值**（递进 / 并列 / 反差 / 引申）：是**Vol.N 与 Vol.N-1 的关系**（是 series_structure 内部的关系描述）
>
> 两者正交：5 个 series_structure × 4 个 relation_to_previous = 20 种组合。例："渐进深入" series + Vol.3 = "递进" relation to Vol.2。
