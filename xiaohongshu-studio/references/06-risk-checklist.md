# 风险提示清单

**这些不是硬伤，但值得告诉用户**——让用户自己拍板要不要改。

## 1. 数据口径风险

**触发条件**：
- 引用的数据存在版本差异（不同书/论文数字不同）
- 数据是估算或"约"值
- 涉及百分比、时间、身份等易被质疑的量化描述

**建议**：加"约"、"据……"、"作者观点"等缓冲词，或直接标注来源。

## 2. 视觉/首图风险

**触发条件**：
- 首图偏深色/冷色（小红书亮色系点击率更高）
- 首图文字过密、留白不足
- 首图颜色对比不够，白天户外看不清

**建议**：给一张亮色系备选首图做 A/B 测试。

## 3. 话题标签风险

**触发条件**：
- 8 个以上标签都是长尾小众（曝光池不够）
- 用了作者名/书名等冷门专有词做主标签
- 标签与内容主题相关性弱

**建议**：至少保 3 个大流量标签在前面。

## 4. 情绪导向风险

**触发条件**：
- 通篇制造焦虑（"再不 XX 就 YY"）
- 贬低某个人群（"什么都不做的人"、"某某专业的人")
- 挑起对立（性别/地域/职业/年龄）

**建议**：把焦虑改成"共鸣 + 解决"结构。

## 5. 内容原创性风险

**触发条件**：
- 直接搬运他人拆书 / 内容
- 使用他人独有的框架但未署名
- 与已有爆款高度相似

**建议**：标注参考来源，或加入个人见解 / 案例。

## 6. 发布时机风险

**触发条件**：
- 与热点话题冲突（同期有大流量新闻）
- 内容涉及节日 / 事件而错过窗口
- 平台正在打压某类内容（如医疗、金融、教培）

**建议**：错峰发布，或调整选题避开风口。

## 输出格式

在审校记录中用以下格式：

```markdown
## 风险提示（N 条）

1. **{风险名}** — {问题描述}
   → 建议：{做法}
   → 是否阻塞：❌ 不阻塞 / ⚠️ 建议改后发 / 🚫 建议不发

2. ...
```

每条不超过 3 行，让用户 30 秒内看完并决策。
