# 正文结构库

## 长版结构（350–450 字）

```
[钩子段 · 2–3 句]
  开门见山抛痛点/反差/悬念
  ↓
[铺垫段 · 3–4 句]
  给一个共鸣场景，让读者对号入座
  ↓
[核心段 · 4–6 句]
  说清楚核心观点/公式/机制
  给至少 1 个具体数据或例子
  ↓
[价值段 · 3–4 句]
  为什么这对你重要
  改变了作者/我什么
  ↓
[预告段 · 1–2 句]
  滑到最后有 action list / 收藏包
  ↓
[尾段 · 1–2 句 + emoji]
  一句情绪呼应封面钩子
```

### 长版示例框架

```
好久没这么想推 X 了 🍂

（钩子 + 情绪）...

📌 书里最戳我的一个 {观点/公式}：
{核心内容}

（价值传递 + 数据支撑）...

作者还提出：
{辅助数据/观点}

所以你不是 {误区}，
你是 {真相} 🌱

我把整篇/整本书的精华拆成了 X 张图，
包括：
· {要点 1}
· {要点 2}
· {要点 3}

滑到最后有可直接抄作业的 action list ✍️

如果你也在 {痛点场景}，
真的建议存一份 👉
```

## 短版结构（120–180 字）

```
[钩子句 · 1 句]
  一句就要能让人停下来
  ↓
[反差数据/公式 · 2–3 句]
  一个数字砸出来
  ↓
[归因/顿悟 · 2 句]
  不是 A，是 B
  ↓
[收藏钩 · 1 句]
  滑到最后 / 自己收藏
```

### 短版示例框架

```
读完这本书，我删掉了 X 个 App 📵

《书名》告诉我一个残酷公式：
{核心公式}

{关键数据}。

不是你 {误区}，
是你 {真相} 🥲

X 张图拆完整本书，
最后一页有今晚就能开始的 X 件事，
自己收藏 🌱
```

## 情绪节奏原则

- 段落之间要有"呼吸"：一句情绪、一句信息，交替进行
- 数据和感受交叉出现，避免连续 3 句都是干货
- Emoji 用在情绪高点，不要每段都放
- 结尾必给一个"下一步动作"（收藏/关注/评论），不要断在信息陈述

## 换行/排版规则

- 手机屏 3–4 行一段（每段约 40–60 字）
- 长句 → 拆成 2–3 行竖排感
- 用 📌 · 💡 ✍️ 🌱 这类符号做视觉锚点，一段 1 个即可
- 避免密集列表，用"，"和换行代替 bullet
