# 小红书爆款内容结构参考

## 四大主流结构

### 1. 痛点解决型（最稳）
**结构**：痛点场景 → 归因 → 方案 → 行动清单
**适合**：干货、方法论、书拆
**首图特征**：深色底 + 大字 hook + 一句悬念

**示例骨架**（9 页）：
- P1：为什么你 XX？（痛点提问）
- P2：三种典型症状（画像共鸣）
- P3：真正的原因（打破误解）
- P4：核心机制/公式（认知升级）
- P5：分类解决方案
- P6：关键数据（信念强化）
- P7：可复用法则
- P8：今晚就能做的 3 件事
- P9：金句 + 关注 CTA

### 2. 反常识型（涨粉快）
**结构**：常见误解 → 反差事实 → 真相 → 行动
**适合**：科普、行业观察、书拆
**首图特征**：亮色底 + "别再 XX 了"式否定钩子

### 3. 干货清单型（收藏率高）
**结构**：X 条法则/工具 → 逐条展开 → 使用场景 → 总结
**适合**：工具集合、书单、清单类内容
**首图特征**：数字 + "亲测有用"/"直接抄"

### 4. 情绪共鸣型（涨互动）
**结构**：故事场景 → 情绪爆点 → 顿悟 → 呼吁
**适合**：心情感悟、观点表达
**首图特征**：文艺色调 + 手写感字体 + 一句心事

## 首图设计三定律

1. **3 秒钩子**：用户滑动时 3 秒内必须看到"关我什么事"
2. **文字要能截图**：主标题字号 ≥ 40pt，深浅对比 ≥ 4.5:1
3. **有情绪 anchor**：颜色、符号（🌱⚡🔔）、emoji 制造情绪指向

## 内容长度对照

| 类型 | 页数 | 正文字数 |
|---|---|---|
| 观点分享 | 3–5 页 | 100–150 字 |
| 拆书/科普 | 8–10 页 | 300–500 字 |
| 深度干货 | 10–15 页 | 500–800 字 |
| 教程手册 | 15+ 页 | 800+ 字 |

默认 9 页是平衡完读率和信息密度的最佳值。

## 话题标签策略

- **大流量池**：3 个（#成长 #读书 #干货 之类）
- **精准标签**：4–6 个（书名、领域、场景）
- **长尾/小众**：2–3 个（细分人群、方法论名）
- **总数**：8–12 个，避免超过 15 个（会被判定为标签堆砌）
