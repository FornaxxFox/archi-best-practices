# 语感转换规则 · 小红书 → 各平台

## 核心原则

**改写不是翻译，是重塑语气 + 重组信息。** 同一观点在 6 个平台呈现 6 种面貌，但内核不变。

## 小红书基线语感

- 朋友聊天感（"我"、"你"、"我们"）
- emoji 节奏：每段 1–2 个
- 短句 + 换行（3–4 行一段）
- 故事 + 干货混合
- 自嘲 / 共鸣 / 焦虑 / 恍然大悟
- 暖调、亲切、不说教

## → 抖音图文

| 维度 | 转换 |
|---|---|
| 字数 | 350→ 280 字（保留 1 个金句 + 1 个反转） |
| emoji | 减少到每段 1 个，关键钩子位置加 1 个 |
| 语气 | 加"反转 / 惊讶 / 挑战"感（"等等"、"不是吧"） |
| 句式 | 多用反问 + 短句（"凭什么？"、"懂了"） |
| 标签 | 换 5–8 个抖音热词（如 #挑战30天 #冷知识 #反常识） |
| CTA | "评论区告诉我" / "下期更狠" |

**避坑**：
- 不要把小红书的"姐妹们"硬搬
- 不要用"亲测有用 / 干货满满"等小红书口头禅
- 不要用小红书语感的"～"（抖音风格更直白）

## → 公众号

| 维度 | 转换 |
|---|---|
| 字数 | 350→ 1500–2000 字（每个观点补论据 + 例子） |
| emoji | 几乎不用（最多 3 个，配图说明用） |
| 语气 | 沉稳 + 结构化（"我们来看" / "具体来说"） |
| 句式 | 长句 + 短句交替，复杂观点用"首先 / 其次 / 最后" |
| 结构 | 加章节小标题（H2 拆 3–5 段） |
| 标签 | 文末加 1–2 个 #话题 或不留 |
| CTA | "关注 / 收藏 / 在看" 三连引导（不强制） |
| 头图 | 必须配 900×383 横幅头图 |

**避坑**：
- 公众号读者对密集 emoji 反感
- 不允许"求转发"等诱导词
- 公众号标题更"信息型"（小红书标题更"情绪型"），保留钩子 + 补信息

**扩写策略**（350→ 1500 字）：
1. 把每个观点拆成"现象 → 原因 → 例子 → 数据" 4 步
2. 加 1 段"作者亲历"或"读者故事"
3. 加 1 段"延伸阅读"（相关书/文章链接）
4. 文末加"关于作者" / "关于我们"

## → 知乎

| 维度 | 转换 |
|---|---|
| 字数 | 350→ 1500–2500 字（必须有论据） |
| emoji | 几乎不用 |
| 语气 | 理性 + 谦虚（"可能"、"一种观点"、"据某研究"） |
| 句式 | 论据 → 推理 → 结论（学术体） |
| 结构 | "问题背景 → 我的回答 → 详细论述 → 总结" |
| 引用 | 至少 1 处论文/报告/权威 |
| 标签 | 选 3–5 个知乎话题（如"心理学"、"个人成长"） |
| 结语 | "以上" + 互动钩（"你的看法？"） |

**避坑**：
- 不允许"家人们"、"绝绝子"、"yyds"
- 知乎用户对"无证据观点"反感，必须补论据
- 引用格式：用 `> 引用` 或截图

**补论据技巧**：
- 找相关研究（如《深度工作》引用 Cal Newport 本人研究）
- 加数据来源（如"据《2024 国民阅读报告》"）
- 加"反方观点 + 反驳"（知乎用户爱看这个）

### 模板示例 · 知乎改写（v1.3.4 新增）

小红书原文（350 字长版）讲"每天 4 小时深度工作 > 8 小时瞎忙"，知乎改写要补论据 + 加反方 + 给文献：

> **问题背景**：
> 有人问：为什么你每天工作 10 小时，产出却不如别人 4 小时？这是知识工作者普遍遇到的悖论。可能原因有：注意力分散、缺乏深度工作能力、被会议和通知打断等。
>
> **我的回答**：
>
> Cal Newport 在《Deep Work》（2016）中提出 4 条核心法则。我用 6 个月时间亲身验证 + 调研了 20+ 高效人士的实践，整理出可操作版本。
>
> ### 法则 1：时间块工作法
>
> 原理：把一天切割成"深度"和"浅度"两段时间。
> > 引用 Cal Newport 本人研究：高质量 4 小时 = 1 个完整工作日
>
> 操作：
> - 早上 9:00–13:00 设为深度块
> - 下午 14:00–18:00 设为浅度块
> - 周末完全断连
>
> ### 法则 2：主动分心
> （详细论述 400 字 + 引用 1 个反方观点）
>
> ### 法则 3：关闭通知
> （详细论述 400 字）
>
> ### 法则 4：仪式感
> （详细论述 400 字）
>
> ## 反方观点 + 反驳
>
> > 反对声音：4 小时工作制不切实际，大多数公司要求 996。
>
> 反驳：Cal Newport 自己作为 MIT 教授 + 畅销书作者 + 4 个孩子的父亲，每天深度工作时间不超过 4 小时。问题不是时间不够，是"看起来很忙"。
>
> ## 我的实证数据
>
> 6 个月实测：
> - 深度工作前：日均 1.5h 深度时间
> - 深度工作后：日均 4.2h 深度时间
> - 周产出：增加 2.3 倍
>
> 以上。你的看法？深度工作 4 小时 = 普通 8 小时，对你适用吗？

**为什么这样写**：
1. **问题背景**（一段）：直接抛悖论，知乎用户爱看"问题导向"
2. **引用研究 + 数据**：Cal Newport 本人研究 / 6 个月实测 / 20+ 高效人士
3. **每条法则扩写到 250-400 字**：补论据、例子、个人故事
4. **反方观点 + 反驳**：知乎特有结构，"我考虑了反对意见"的姿态加分
5. **实证数据**：6 个月 before/after 对比，有数字
6. **"以上"结语 + 互动钩**：知乎惯例

## → 即刻

| 维度 | 转换 |
|---|---|
| 字数 | 350→ 200–280 字（删到 1 句金句 + 1 段思考） |
| emoji | 1–2 个，不要小红书风格的多 |
| 语气 | 思考 + 自嘲 + 文艺 |
| 句式 | 一行一个意思，多用"——"分隔 |
| 标签 | 1–2 个 #（如 #深度工作 #思考） |

**避坑**：
- 即刻用户爱"反转 + 思考"，不要纯干货
- 不要用"求赞"、"求关注"等小红书语
- 短句节奏优先，长句会被跳过

**模板示例**：
```
我之前以为深度工作 = 远离手机。
直到读了《深度工作》第 3 章才发现：
真正的高手用"主动分心"做深度工作 —— 提前留出低质量时间，
剩下的时间反而都高质量。

一个公式分享：
低质量时间（先标记）+ 高质量时间 = 真正的深度工作

你呢？你怎么分配？
```

## → Twitter (X)

| 维度 | 转换 |
|---|---|
| 字数 | 中文 ≤ 280 字 / 英文 ≤ 280 字符 |
| emoji | 0–1 个，节制 |
| 语气 | 直接 + 强观点 + 自嘲（允许） |
| 句式 | 1 个核心 hook + 1 个数据 + 1 个 CTA |
| 标签 | 1–2 个英文 # |
| 配图 | 单图（推荐 16:9） |

**避坑**：
- 不允许"求转发 / 求点赞"
- 标签必须英文（如 #productivity #focus）
- 1 条 thread = 1 张图 + 1 段话；长内容用 thread 串

**模板示例**（英文）：
```
Most people think "deep work" = no phone.

Wrong.

The real hack: schedule your shallow time first.
Everything left = deep time.

4h of deep work > 8h of "looking busy"

— via Cal Newport
```

- [ ] 字数在 ±10% 上限内
